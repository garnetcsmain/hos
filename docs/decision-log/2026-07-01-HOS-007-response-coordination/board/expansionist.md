# Expansionist Review: HOS-2026-007

## My Recommendation
🟡 RESHAPE FOR MORE UPSIDE

## Biggest Upside
The reunification matcher is the wedge; coordination is the platform. If HOS
becomes the shared, auditable operational picture for a disaster — the place
where shelters, needs, and supplies reconcile — it stops being an app and
becomes the **coordination backbone** every responder plugs into. Scale: from one
Venezuela incident to the standard substrate any multi-agency response adopts.
Timeline: the wedge earns the right; the backbone compounds over years.

## Domino Effects
1. Shelter capacity + needs board (this slice) -> the first shared source of
   truth agencies check instead of WhatsApp -> HOS becomes the default board.
2. A needs<->supplies ledger -> a real-time picture of unmet need by district ->
   funders and logistics route against HOS data -> HOS becomes the reporting layer.
3. Reunification + coordination on ONE event store -> "this found person is now
   sheltered here, and their family's need is water" — the family graph and the
   ops graph reconcile, which no single-purpose tool can do.

## Moat Opportunity
The compounding asset is the **append-only, cross-org event store as a neutral
standard** — an auditable shared ledger of who-needed-what-when and who-served-it.
Whoever holds the trusted, neutral, offline-first record of a response becomes
hard to displace, the way PFIF/Person Finder became the interchange for missing
persons. Trust + audit + interoperability is the moat, not features.

## Adjacent Opportunities
- Other crisis types (flood, conflict displacement) reuse the same needs/supplies primitives.
- A read-only, aggregated, de-identified feed for funders/UN clusters (situational awareness) without exposing sites.
- An interop export (open schema) so other tools ingest HOS's needs ledger — seed the standard early.

## Resource Bottleneck
Secure the **shared org/actor identity + event schema now, while the data is
small.** The board already asked for real org/actor entities (D2/D3). If
coordination is bolted on without them, every cross-org feature later is a
painful retrofit. Model the org graph and a versioned coordination event schema
in this slice even if the UI is thin — that is the cheap-now, expensive-later input.

## Confidence Score
0.7 — the upside is real and the wedge->platform logic is sound. Not higher
because the moat only exists if reunification actually ships and earns the trust
that makes agencies adopt the board; a half-finished mission caps the upside.

## What I Don't Know
Whether agencies will publish real capacity/needs into a neutral system rather
than hoard it — adoption, not tech, is the binding constraint (Researcher should test this).
